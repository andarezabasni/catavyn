# Contributing

Thanks for wanting to help. Here's how to get started.

## Setup

Follow the local setup in the README to get the app running. You'll need your own Supabase project for development.

## Making changes

1. Fork the repo and create a branch from `main`
2. Make your changes
3. Make sure `npm run build` passes before submitting
4. Open a pull request — describe what you changed and why

## What's welcome

- Bug fixes
- UI/UX improvements that stay within the existing design language
- Performance improvements
- New features that fit the core scope (notes + tasks, no bloat)

## What to avoid

- Adding new dependencies without a good reason
- Changing the color palette or typography without discussion
- Features that belong in a heavier tool like Notion

## Questions

Open an issue if you're unsure about something before spending time on it.
